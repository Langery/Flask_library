## 2026-06-12 X-A 组:Flask 后台抽公共层 + 重组 blueprint + 瘦身 server.py(单 commit)

**目的**:Flask 后台当前 350 行,9 个小文件,提供 6 个业务 endpoint。代码结构有几处明显问题:① 类包装冗余(9 个类大多只暴露 1-2 个方法,`__init__` 干的事都是 `print + 读 request data`)② `print('success use xxx')` 噪音(每次请求 1 行无信息 stdout)③ `dataLab.py` 60 行 SQL 拼接(没被用,但留隐患)④ 两个空壳 blueprint 占 url_prefix ⑤ server.py 111 行里杂事太多 ⑥ row_factory 设置了但 list_table 还在用 tuple 索引。本轮重组结构,统一基础设施,删死代码,前端 0 影响(路由 prefix 保持不变)。

**改动范围**:16 文件(9 删 + 6 新 + 1 改 + 1 gitignore 改)

#### 新建 `classStore/common/`(4 文件 + 1 `__init__.py`)

| 文件 | 内容 | 来源 |
|------|------|------|
| `__init__.py` | 空包标记 | 新 |
| `db.py`(~75 行) | `get_conn()` 单例 + `query_all/sql, params` / `query_one/sql, params` / `execute/sql, params` 三个 helper,统一 `row_factory=sqlite3.Row`(用 `item['col']` 而非 `item[0]`);`init_db()` 启动建表(已有 IF NOT EXISTS) | 抽自 `classStore/server/__init__.py` 的 POOL/SQLitePool |
| `response.py`(~15 行) | `ok(data, message='success', http_status=200)` / `fail(message, code=1, http_status=400, data=None)` 统一 `{code, message, data}` 格式 | 新 |
| `errors.py`(~25 行) | `register_error_handlers(app)` 注册 404/405/HTTPException/Exception 兜底,500 走 `logger.exception` 记录 | 新 |
| `auth.py`(空) | 占位,X-B 真正实现 | 新 |

#### `server.py` 瘦身:111 行 → 45 行

**变化**:
- 删 4 个 `app.route(...)` → 改 2 个 `app.register_blueprint(...)`
- 删 6 个 `from classStore.requestway...` import → 0 业务 import
- 删 SQLConfig 配置 / `getPool.config(SQLConfig)` / 6 个 print debug
- 启 `init_db()`(自动建表)替代手写 DDL 在 POOL 类里
- 注册 `register_error_handlers(app)`

**保留**:
- `app.run(host='127.0.0.1', port=5000)`(原值)
- CORS 配 `supports_credentials=True`
- logging 配 FileHandler + Formatter(原值)
- 路由 prefix 保持**根级**(`/login` `/register` 等),蓝图注册时不带 `url_prefix`,**前端 axios.baseURL = `http://127.0.0.1:5000/` 0 改动**

#### 删 6 个类文件 + 1 个隐患文件

- `classStore/requestway/first.py` / `login.py` / `register.py` / `talk.py`(被新 auth/routes.py 合并)
- `classStore/requestway/pages_server/list_table.py` / `upload_images.py`(被新 pages/routes.py 合并)
- `classStore/server/dataLab.py`(60 行 SQL 字符串拼接,`like()` 等方法没人调,有 SQL 注入隐患,留之无用)

#### `.gitignore` 加 `flask.db` + `*.sqlite3`

之前 `flask.db` 没被 ignore,导致每次启动都出现 untracked 文件。本轮加进去,数据库本地生成不污染 VCS。

### 路由 prefix 兼容性

| 接口 | 原 prefix | 新 prefix | 前端调用 |
|------|-----------|-----------|----------|
| `/first` | 根 | 根(blueprint 无 url_prefix) | `api.get('first', { params: { name } })` |
| `/login` | 根 | 根 | `api.post('login', params)` |
| `/register` | 根 | 根 | `api.post('register', params)` |
| `/getTree` | 根 | 根 | `api.post('getTree', ...)` |
| `/getListInfor` | 根 | 根 | `api.get('getListInfor', { params: { id } })` |
| `/uploadImg` | 根 | 根 | `api.post('uploadImg', data)` |

**前端 0 改动**。axios.baseURL `http://127.0.0.1:5000/` 拼接每个 path,直达 Flask。

### 不在本轮范围

- **JWT 真签名 + verify 装饰器**:留到 X-B(本 commit 保持原简化手搓 JWT,5 分钟 exp,空签名)
- **新业务接口**(library/layout/news):留到第 2 批 X-C/D/E
- **calendar 系列 endpoint**:`POST /calendar/list` 和 `POST /canlendar` 已被前端 `indexview.js` 调用,但当前**未在 server.py 注册**(会 404)。原状态如此,本轮不修(避免 scope 蔓延;calendar 是 `temporary_storage/` 里的 WIP,有专门 plan)
- **listtable 表结构**(只有 3 列 id/name/describeInFor,无 parent_id):原 `getTree` 用 `item[3]` 越界,本身就有 bug;本轮只做结构搬运,**不修逻辑 bug**

### 风险

- 6 个类文件全删,如果后端还有别的脚本 import 它们(在 Flask_library 项目内)会立刻挂
  - 验证:`grep -rn "from classStore.requestway\|classStore.requestway.first\|...requestway.login\|...requestway.register" Flask_library/` → 无残留(除了已删文件自身)
- `dataLab.py` 删除,如果有任何代码依赖它的 `SQLFun` 类也会挂
  - 验证:整个项目 grep "dataLab" / "SQLFun" → 无调用
- 路由 prefix 不变,前端 0 影响,但**确认前端没在任何路径里硬编码 `auth/login` 这种带 prefix 的 URL**:已 grep `api.\(get\|post\)` → 6 个调用都是裸路径,安全
